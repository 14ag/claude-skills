# Writing Standards

Use this reference for any substantial prose generated by the skill.

## Audience Levels

Match the language level to the document:

- `B2` for user-facing sections such as README introductions, quick starts, and
  troubleshooting steps
- `C1` for contributor guides, architecture notes, API references, and detailed
  development instructions

## Style Rules

- Be direct, concrete, and specific.
- Prefer plain verbs like `use`, `run`, `check`, and `update`.
- Keep sentences tight unless added detail improves accuracy.
- Write like an experienced engineer addressing another engineer.
- Prefer evidence over marketing language.

## Avoid AI-Tell Language

Do not use these words or phrases unless a precise technical meaning truly
requires them:

- `delve`
- `tapestry`
- `landscape` as vague metaphor
- `leverage`
- `utilize`
- `crucial`
- `pivotal`
- `robust` as filler
- `seamless`
- `foster`
- `empower`
- `underscore`
- `at its core`
- `it is important to note`

Prefer plain alternatives such as `use`, `important`, `reliable`, `support`,
`highlight`, or simply restructuring the sentence.

## Content Constraints

1. Ground technical claims in the repository or supplied context.
2. Omit unsupported sections rather than filling them with placeholders.
3. Ask at most one consolidated clarification question when critical details are
   missing.
4. Keep file operations idempotent when editing existing docs.
5. Do not overwrite human-written content unless the request is explicitly to
   revise it.

## Structure Guidance

- Use headings only when they help the reader scan the output.
- Avoid empty summaries and generic conclusions.
- Do not pad short content to fit a template.
- For procedural content, prefer ordered steps.
- For comparative or evaluative content, make the criteria explicit.

## Source Handling

When external standards shape the output, cite them in the reference-driven
workflow or in the generated document if the format calls for it.

Do not cite standards to compensate for missing repository facts.
